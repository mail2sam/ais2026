01 / Foundation

The conversation has changed.

The winning question is no longer "what is the best prompt?" It is "which parts of the practice should become AI-assisted workflows?" A workflow reads documents, retrieves authoritative sources, runs deterministic checks, and prepares structured output for partner review.

The prompt mindset
"How do I write a better prompt?"

One input, one output, one person getting faster inside one chat window.

The workflow mindset
"Which work should become repeatable?"

A chain of steps the whole firm can run consistently, with audit trail and human judgment intact.

Read

Intake and reconcile

Parse notices, ITRs, TARs, AIS files, ledgers, and GST reports. Surface the deltas before drafting begins.

Retrieve

Ground every claim

Use Perplexity, NotebookLM, and curated libraries to anchor provisions, circulars, standards, and case law.

Draft

Review before filing

Produce submissions, annexure lists, working-paper notes, and a partner checklist with citations marked for verification.

Use
Read this like a build guide. Pick one workflow, assemble the smallest tool chain, ship the first working version, and only then add automation or integrations.

03 / CA Workflows

Deployable workflows for core practice work.

Each workflow has a small stack, a repeatable sequence, and a review point. Use the structure, then adapt the prompts to your firm style.

Workflow 01

Income Tax Scrutiny Reply

Inputs: Notice, ITR, TAR, AIS, ledgersOutput: Draft submission with annexures
Claude ProjectNotebookLMPerplexity
  1. Create the projectLoad standard instructions: senior tax counsel, no invented citations, mark every uncertain claim with [VERIFY].
  2. Attach the file setNotice, ITR, TAR, AIS, ledger extracts, and previous submissions.
  3. Prepare the issue tableAsk for section, item, department allegation, assessee position, evidence, and pending documents.
  4. Retrieve law and casesUse NotebookLM for your library and Perplexity for recent circulars or rulings.
  5. Draft and reviewGenerate the reply, annexure index, and partner checklist before filing.
Workflow 02

GST Notice Reply

Inputs: SCN, GSTR returns, 2B, PR, e-way dataOutput: Tiered defence with reconciliation
Claude GST ProjectBolt dashboardNotebookLM
  1. Classify the issueITC mismatch, outward tax mismatch, registration, e-way bill, classification, or interest and penalty.
  2. Reconcile numbersCompare 2B, purchase register, vendor status, and return data with tolerance thresholds.
  3. Build the defencePrimary facts, alternative law, and without-prejudice quantum computation.
  4. Verify citationsConfirm every circular, section, and ruling on official sources before sending.
Workflow 03

Statutory Audit Working Paper Review

Inputs: Trial balance, lead schedules, SA checklistOutput: Exceptions and review notes
ClaudeNotebookLMReplit Agent
  1. Load standardsCreate a NotebookLM library for SAs, Schedule III, CARO, and firm audit templates.
  2. Prepare dataUse Replit Agent for repeatable CSV or Excel cleaning where formulas matter.
  3. Ask for exceptionsSurface missing evidence, unusual ratios, stale balances, and unlinked schedules.
  4. Write review notesConvert findings into partner review points and client query lists.

05 / Vibe Coding Lab

Type a sentence. Get an app.

These are real platform-specific build prompts. Use them as starting points, then iterate with changes like "add export", "tighten validation", or "make the admin view partner-only."

Caution
Prototype first, harden before client data. Vibe-coded apps need authentication, audit logging, access control, and security review before handling client PII.

lovable.dev

Best for client portals and forms

Use Lovable when you want a real app quickly: onboarding forms, dashboards, authenticated portals, and simple workflows with persistence.

Prompt / client portals and forms

CA use case: GST reconciliation dashboard for GSTR-2B and purchase register uploads.

Prompt / CSV, Excel, and dashboard logic
Build a GST reconciliation web app.

UI: two upload zones. Left: GSTR-2B CSV. Right: Purchase Register CSV. Match on GSTIN + invoice number + invoice month with small rounding tolerance.

Output four buckets: Matched, Mismatched, In 2B not in PR, In PR not in 2B. Show metric cards, sortable tables, vendor-flag checkboxes, and an Excel mismatch report using SheetJS.

Replit Agent

Backend-first scripts / scheduled jobs / cloud runtime
replit.com

Best for backend automation

Use Replit Agent when the deliverable is a job, not a UI: file watchers, scheduled digests, scrapers, and recurring reconciliations.

CA use case: Google Drive document classifier and filer.

MVP30 minutes
SkillLight Python
Prompt / backend automation
Build a Python script that watches a Google Drive folder called Inbox and automatically classifies and files documents.

For each new file, read filename and extracted text. Call Claude API to return strict JSON: client_name, document_type, period, confidence.

Use clients.xlsx as the canonical client folder map. Rename and move high-confidence files into /clients/{Folder}/{Year}/{DocType}/. Move low-confidence or unmatched files to /_review. Log every action to a Google Sheet.

Run on a 5-minute schedule in Replit. Add error handling and a daily summary email.

06 / Roadmap

What to build, in what order.

The sequence below keeps the firm from trying to AI-enable everything at once.

Week 1

Get the beginner stack live

ChatGPT Plus, Claude Pro, and Perplexity Pro. Use all three on live work to learn their strengths.

Week 2

Build your first Custom GPT

Pick one recurring workflow and encode house style, jurisdiction, structure, and verification rules.

Month 1

Move to Projects and Skills

Convert the highest-volume workflow into a Project with reference documents and deterministic scripts.

Month 2

Create the knowledge base

Use NotebookLM for Acts, Rules, standards, circulars, and curated case-law libraries.

Month 3

Vibe-code one internal app

Choose a notice tracker, onboarding form, GST recon dashboard, or filing automation.

Months 4 to 6

Add automation and MIS

Wire n8n or Make across forms, Drive, Sheets, email, and the AI stack.

07 / Governance

Scale with discipline.

AI in a CA firm is a regulated professional workflow. Treat confidentiality, citations, consent, and review as product requirements.

DPDP Act 2023

Client data is personal data

Document consent, disclose processing, and avoid uploading sensitive PII to consumer tools without explicit approval.

ICAI Ethics

Confidentiality is non-delegable

Use approved Pro or business plans for client-identifying material and maintain firm-level tool policies.

Judgment

AI is a junior, not a co-signatory

Every filing, opinion, and client deliverable still needs CA review and professional skepticism.

Citations

Verify every authority

Case law, section references, circulars, and quoted standards must be checked on official sources.

Audit Trail

Document AI assistance

Keep prompt, model, date, reviewer, and material changes in the working paper trail.

Vendor Risk

Read platform terms

Review data processing, retention, training use, residency, and enterprise controls before firm-wide rollout.
